# 정책 데이터 → 정부 표준 슬라이드 — 파이프라인 실습

> 공개 정책 데이터(대한민국 AI 행동계획 2026)를 수집·정규화하여 엑셀 현황판을 갱신하고,
> **대한민국 정부 디자인 표준(KRDS) 테마의 보고용 슬라이드**를 자동 생성하는
> 실습입니다. Codex 대화형 세션 하나에서 **수집 → 갱신 → 디자인 생성 → 검수**의
> 파이프라인형 멀티 스텝 작업을 서브에이전트 분업으로 지휘합니다.

## 1. 문제 정의

정책 이행현황 보고는 전형적인 반복 지식노동입니다. ① 흩어진 원천 데이터(공식 문서 + 분석 대시보드)를 수집하고, ② 기준일에 맞춰 현황 지표를 재계산하고, ③ 조직 표준 디자인의 슬라이드로 재작성합니다. 매주·매월 같은 구조가 반복되지만, 데이터 소스가 비정형(웹 대시보드, PDF, JS 임베디드 JSON)이라 단순 매크로로는 자동화가 어렵습니다.

이 실습은 그 전 과정을 **"역할(누가) + 지식(어떻게)" 분리 원칙의 하네스형 시스템**으로 구성합니다. `xlsx/` 실습이 단일 프롬프트 ETL이었다면, 이번에는 수집 → 갱신 → 디자인 생성 → 검수로 이어지는 **파이프라인형 멀티 스텝 작업**입니다. 여러분은 실무자를 넘어, 단계마다 다른 전문가에게 일을 나눠 주고 산출물을 다음 단계로 넘기는 **팀장 역할**을 Codex와 함께 수행합니다.

## 4. 파일 구성

| 파일 | 역할 |
|---|---|
| `AI행동계획.xlsx` | **데이터 계층**. `★최종안` 시트(326개 권고 원문) + 이행현황 컬럼(기한일자·D-Day·위험도·이행상태, 전부 **수식**) + `이행현황요약` 시트(기준일 셀 기반 자동 집계) |
| `AI행동계획_이행현황.pptx` | **산출물 예시**. KRDS 테마 9장 보고 슬라이드 (기준일 2026. 7. 3.) |
| `README.md` | 본 교안 |

핵심 설계 — **기준일 하나만 바꾸면 전체가 갱신되는 살아있는 문서**: `이행현황요약!B3`(기준일)을 수정하면 326건 각각의 D-Day와 위험도(기한초과/긴급/주의/관찰/여유), 모든 집계가 수식으로 재계산됩니다. 에이전트가 값을 하드코딩했다면 매번 재작업이 필요했을 것입니다.

## 5. 파이프라인 설계 (역할 × 지식)

이 과정을 관통하는 원칙대로 **역할 정의(누가)와 도메인 지식(어떻게)을 분리**하고, 네 역할을 **파이프라인 패턴**으로 연결합니다.

```
[data-collector] ──▶ [xlsx-updater] ──▶ [slide-designer] ──▶ [qa-reviewer]
     수집·정규화          현황 재계산           KRDS 슬라이드 생성        검수·교차검증
```

| 역할 | 임무 | 작업 방식 |
|---|---|---|
| `data-collector` | 대시보드/저장소에서 과제·권고 데이터 추출, 스키마 검증 (99과제/326권고 카운트 일치 확인) | 웹 데이터 수집 + 파싱 |
| `xlsx-updater` | 기준일 기반 D-Day·위험도 갱신, 요약 집계 추출, 수식 오류 0건 검증 | Python `openpyxl` |
| `slide-designer` | 요약 데이터를 KRDS 테마 슬라이드로 변환 (차트는 이미지가 아닌 네이티브 차트) | Python `python-pptx` + KRDS 디자인 규칙 |
| `qa-reviewer` | 슬라이드 요소 배치 점검(오버플로·겹침), 데이터-슬라이드 수치 교차 검증 | `python-pptx`로 좌표·수치 대조 |

네 역할은 대화형 세션에서 **서브에이전트**로 생성됩니다. 서브에이전트끼리는 직접 대화하지 못하므로, 부모(여러분과 대화 중인 Codex)가 각 단계의 산출물을 확인하고 다음 단계로 넘깁니다. `qa-reviewer`가 결함을 발견하면 부모가 그 목록을 받아 `slide-designer` 단계로 되돌리는 **수정 루프**도 부모가 조율합니다.

### KRDS 디자인 규칙 (slide-designer 단계에 주입할 지식)

- **팔레트**: Primary `#246BEB`(KRDS 파랑) 지배 + 심연 남색 `#0B2E5C`(표지·클로징) + 위험 `#DE3412` / 주의 앰버 / 여유 초록은 상태 표시에만 제한 사용
- **구조**: 다크 표지 → 라이트 본문 → 다크 클로징 (샌드위치), 좌상단 제목 + 한 줄 핵심 메시지 고정
- **데이터 우선**: 큰 숫자 스탯 카드 + 네이티브 차트(도넛·막대·누적바), 표는 남색 헤더
- **정직성 장치**: 모든 장에 기준일 푸터, ‘기한초과 ≠ 미이행’ 해석 주의 문구, 클로징에 출처·한계 명시

이 규칙 블록이 곧 **외부화된 도메인 지식**입니다. 이번 실습에서는 프롬프트에 직접 넣어 실행하고, 심화 과제에서 `.agents/skills/` 스킬 파일로 굳혀 봅니다.

## 6. 실행 — 대화형 세션에서 파이프라인 지휘

`codex`를 실행하고 아래 프롬프트를 그대로 입력합니다. Codex가 파일 수정·명령 실행(pip 설치, 웹 데이터 수집 등) 전에 **승인을 묻습니다**. 내용을 확인하고 허용하세요.

## 7. 관찰 포인트

- **부모가 팀장입니다.** 서브에이전트끼리는 대화하지 못하므로, 수집 노트(`collect/summary.md`)라는 **합의된 산출물 형식**이 단계 사이의 공용어가 됩니다 (원리 3: 계약이 병렬성·반복성을 만든다).
- **갱신 단계에서 수식을 값으로 덮어쓰지 않는지** 확인하세요. 엑셀을 다시 열면 바뀐 기준일로 전체가 재계산되는 "살아있는 문서"가 유지되어야 합니다 (원리 2: 지식의 외부화 — 로직을 수식으로 파일에 남김).
- **검수는 만든 에이전트가 하지 않습니다.** slide-designer의 결과를 qa-reviewer가 좌표·수치로 대조합니다 (원리 4: 검증자 분리).
- **KRDS 규칙 블록은 프롬프트에서 분리 가능한 지식**입니다. 규칙만 바꾸면 같은 파이프라인이 다른 조직 표준의 슬라이드를 냅니다 (원리 1: What·Who·How 분리).

## 8. 검수 체크리스트

- [ ] 엑셀 수식이 값으로 덮어써지지 않았고, 엑셀에서 열면 기준일 변경분이 자동 재계산되는가
- [ ] 슬라이드 수치 = 집계값 (권고 326 / 위험도 합계 326 / 정책축 134+114+78)
- [ ] 차트가 이미지가 아닌 편집 가능한 네이티브 차트인가
- [ ] KRDS 팔레트 준수 (파랑 지배, 상태색 제한 사용, 다크-라이트-다크 구조)
- [ ] ‘기한초과 ≠ 미이행’ 해석 주의 문구와 UNOFFICIAL 출처 명시 여부
- [ ] 텍스트 오버플로·요소 겹침 없음 (qa-reviewer 좌표 점검 기준)

## 9. 교육 포인트

**① 데이터 계층과 표현 계층의 분리.** 엑셀이 SSOT(단일 진실 공급원), 슬라이드는 뷰(View)입니다. 에이전트에게 "슬라이드 수치는 반드시 엑셀에서"라는 불변 규칙을 주면, LLM이 그럴듯한 숫자를 지어내는 것을 구조적으로 차단합니다.

**② 수식은 에이전트가 남기는 재사용 가능한 로직.** D-Day·위험도를 값이 아닌 수식으로 남겼기 때문에, 다음 보고 주기에는 기준일 셀 하나만 바꾸면 됩니다. 에이전트의 산출물이 '일회성 답변'이 아니라 '갱신 가능한 시스템'이 되는 지점입니다. openpyxl이 수식을 계산하지 못한다는 제약조차 교육적입니다 — 도구의 한계를 알면 "수식은 보존, 집계는 별도 계산"으로 역할을 나눌 수 있습니다.

**③ 도메인 지식의 외부화.** KRDS 팔레트, 위험도 분류 기준(D<0/≤34/≤120/≤240), 해석 주의 문구 같은 도메인 규칙을 프롬프트 밖 스킬 문서로 꺼내 두면, 어떤 에이전트가 실행해도 같은 품질이 재현됩니다 — `xlsx/` 실습의 표준용어집과 같은 원리입니다.

**④ 검증 없는 자동화는 자동화된 오보.** 정책 수치가 틀린 보고서는 없느니만 못합니다. 수집 단계의 카운트 일치 검증, 갱신 단계의 수식 오류 검증, 검수 단계의 수치 교차 검증 — 3중 게이트가 파이프라인의 신뢰를 만듭니다.

2. **주기 실행**: 공식 이행실적이 공개되면 `이행상태` 컬럼을 실적 데이터와 연동하고, 위의 `codex exec` 한 줄을 GitHub Actions cron에 등록해 매주 기준일을 갱신하는 배치 잡으로 발전시켜 보세요.
3. **부처별 개별 보고**: "부처별 1장 요약 슬라이드를 담당하는 에이전트들을 병렬로 생성하라"는 팬아웃 지시로 부처별 요약 18종을 병렬 생성해 보세요.
4. **xlsx 실습과 연결**: `xlsx/`의 표준용어집 패턴을 이 도메인에 적용해 '정책 용어 표준화 스킬'을 추가해 보세요.
