모든 실습은 Codex CLI(터미널에서 codex 실행) 또는 Codex 앱에서 '대화형'으로 진행합니다.

실습1. 보도자료 기반 예상 Q&A 생성기
# 보도자료 파일을 둔 폴더에서 codex 실행 후 입력:
 → 이 폴더의 보도자료를 기반으로 예상 Q&A 문서를 만들려고 합니다. 질문 발굴 /
   답변 작성 / 검증 역할의 에이전트 팀을 설계해 순서대로 실행하고, 답변마다
   보도자료의 근거 문장을 붙이세요. 근거가 없는 답변은 [확인 필요]로 표시하세요.

실습2. XLSX 파일 정리하기 (xlsx/ 응용)
# xlsx 디렉토리에서 codex 실행 후 입력:
 → 생산공정의 공정 보고서들을 공정_표준용어집.xlsx 의 표준 공정 용어에 맞춰
   수정·재작성하고, 전체 요약 보고서를 만들어 주세요. 수치는 원본과 100% 일치해야
   하고, 합계·불량률은 수식으로 계산하세요. 용어집에 없는 표현은 [미확인 용어]로
   보고하세요.

실습3. 신규 브랜드를 위한 랜딩 페이지
 → 새롭게 리뉴얼한 '새로' 소주의 브랜드 랜딩 페이지를 만들고 싶습니다. 먼저 페이지
   구성·카피·디자인 토큰(색상 hex·타이포)을 정한 스펙 문서를 작성해 나에게 확인받고,
   승인 후 스펙대로 단일 페이지(index.html)를 구현하세요. 구현 후에는 별도 검증
   단계에서 스펙 대비 누락을 점검하세요.

실습4. 최신 현황 업데이트 및 슬라이드 만들기 (slide/ 응용)
# slide 디렉토리에서 codex 실행 후 입력:
 → "국가인공지능전략위원회" AI행동계획의 실행 현황을 오늘 기준으로 업데이트하고,
   관련한 공유 슬라이드 자료를 만들고 싶습니다. slide/README.md 6절의 파이프라인
   (수집 → 갱신 → KRDS 슬라이드 생성 → 검수)을 서브에이전트 분업으로 실행하세요.
   엑셀 수식은 값으로 덮어쓰지 말고 보존하세요.

실습5. 한국디스플레이 라인 공정설비 대시보드 만들기 (dashboard/ 응용)
# dashboard 디렉토리에서 codex 실행 후 입력:
 → 한국디스플레이의 200대 자동화생산설비 텔레메트리를 실시간으로 보여주는
   대시보드를 만들어 주세요. 먼저 표준 스키마와 API 계약을 contracts/ 에 확정한 뒤,
   시뮬레이터 · 정규화 어댑터 · 대시보드 UI 를 병렬 서브에이전트로 구현하고,
   별도 qa-validator 에이전트로 원시값=정규화값=화면값 3점 교차 검증까지 하세요.

# 먼저 사람이 현재 상태 확인 (codex-engineering/track-a-engineering 디렉토리)
python3 -m pytest tests/ -q        # 초록불 확인 (통과하지만 결함이 숨어 있음)
python3 src/quality_check.py       # B·C 라인은 임계치 초과 위험 수준인데도 '정상' 표시
# 이 폴더에서 codex 실행(또는 Codex 앱에서 폴더 열기) 후 순서대로 입력:
 → src/quality_check.py 를 검토해줘. 불량률(ppm) 계산과 경계 처리에 결함이 있는지 진단만 해줘(수정 금지).
 → 방금 진단한 결함을 수정하고, tests/에 절대값·경계값 테스트를 추가해 강화한 뒤 pytest로 통과 확인해줘. (수정·실행 전 승인 요청이 뜨면 확인 후 허용)
 → 방금 수정을 리뷰어 관점에서 비판적으로 다시 검토해줘.

실습8. Codex 하네스 엔지니어링 — 트랙 B (비개발 업무 자동화)
# codex-engineering/track-b-ops 디렉토리에서 codex --search 실행(또는 Codex 앱) 후 입력:
 → 최근 1주 한국 금융 규제·공시 동향을 조사해줘. 자본규제·소비자보호·AML/제재·공시 4개 영역, 각 항목에 출처·발표일. 확인 안 된 건 [확인 필요]로.
 → 위 내용을 표로 정리해줘. 컬럼: 제목 / 출처 / 발표일 / 분류 / 영향도 / 다음 행동. (여러 번 해보면 형식이 흔들리는 걸 관찰)
# (형식 보장·자동화) 매주 같은 구조를 강제하려면 codex exec + 스키마:
codex exec --search --sandbox read-only --output-schema schema/regwatch.schema.json --output-last-message regwatch.json "최근 1주 한국 금융 규제·공시를 조사해 schema에 맞춰 JSON으로 반환. category·impact는 스키마 값만."

실습9. 오픈소스 MCP Tool 만들기 (대화형·자동화 공용)
# tool/mcp-server 디렉토리에서 서버 준비
pip3 install -r requirements.txt
python3 -c "from server import compute_oee; print(compute_oee('B','2026-07-14'))"   # 언제나 69.8%
# Codex에 등록 (절대경로 권장, 상세는 tool/register-codex.md):
codex mcp add production-metrics -- python3 <절대경로>/codex-edu/tool/mcp-server/server.py
codex mcp list
# codex 대화형 실행 후 입력:
 → production-metrics 도구로 07-14 A·B·C 라인 OEE를 구하고 가장 낮은 라인의 병목을 정리해줘
# 같은 도구를 자동화(codex exec)에서도 — 읽기 전용이므로 read-only 샌드박스:
codex exec --sandbox read-only "production-metrics MCP로 07-14 B라인 OEE와 불량률(ppm)을 구하고 개선 우선순위를 제안해라"
